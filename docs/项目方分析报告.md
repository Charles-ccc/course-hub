## 一、项目全景

**网课超市**（`wangke-supermarket`）是一个"先学后付"在线课程撮合平台。其核心模式是：

- 学员无需先全额付款，签约后按分期还款；
- 课程机构是合同甲方、卖方、收款方，自担逾期与坏账风险；
- 平台定位为撮合 + 技术 + 获客中介，仅向机构收服务费，不归集学员资金、不垫资。

整个工程以 **pnpm workspaces + Turborepo** 管理，包含 5 个可独立部署的应用子项目和 1 个共享包。

## 三、各子项目详解

### 3.1 `apps/api` — 后端服务

| 项         | 内容                                                 |
| ---------- | ---------------------------------------------------- |
| 技术栈     | NestJS 10 · Prisma 5 · MySQL 8 · Redis 7 · Bull 队列 |
| 运行方式   | `pnpm dev:api`（nest start --watch）                 |
| 数据库访问 | Prisma ORM，迁移文件位于 `prisma/migrations/`        |

**功能模块（14 个）：**

| 模块           | 职责                                                          |
| -------------- | ------------------------------------------------------------- |
| `auth`         | JWT 鉴权、登录、令牌刷新，统一身份入口                        |
| `user`         | 学员账户管理、实名核验、OpenID 绑定                           |
| `org`          | 机构入驻、审核、保证金台账、结算费率配置                      |
| `course`       | 课程发布、上下架、分期方案挂载、视频资产管理                  |
| `order`        | 订单创建与全生命周期状态机（创建→冷静期→履约→完成/退款/终止） |
| `installment`  | 分期计划生成、每期内容交付前置校验、代扣指令发起              |
| `credit`       | 信用授权管理、芝麻先享 / 微信支付分接入适配                   |
| `contract`     | 电子合同生成与签署（对接法大大 / e签宝），合同甲方为机构      |
| `commission`   | 业务员提成核算（单级）、提成看板数据                          |
| `referral`     | 学员邀请拉新奖励（单级），归属关系注册时锁定                  |
| `learning`     | 学习打卡、课程内容解锁记录（联动分期交付确认）                |
| `settlement`   | 机构服务费结算周期对账、出账                                  |
| `notification` | 短信 / 小程序模板消息推送（还款提醒、逾期通知等）             |
| `report`       | 平台运营报表、机构数据看板数据聚合                            |

**核心数据模型（Prisma Schema 中的主要实体）：**

```
Org ──< Course ──< Order ──< InstallmentItem ──< Deduction
                    │
                    ├──< Contract
                    ├──< Commission
                    ├──< CreditDecision / CreditAgreement
                    └── Student >── Staff（归属）
```

**基础设施（docker-compose）：**

- MySQL 8.0（端口 3306，数据库 `wangke`）
- Redis 7 Alpine（端口 6379，AOF 持久化）
- Adminer（数据库 Web 管理界面）

### 3.6 `packages/shared-types` — 跨端共享类型

| 项       | 内容                                                              |
| -------- | ----------------------------------------------------------------- |
| 包名     | `@wangke/shared-types`                                            |
| 被引用方 | api / mp-student / mp-staff / web-admin / web-institution（全部） |

统一定义全局使用的 TypeScript 枚举，确保各端状态字面量一致：

- `OrgStatus`（PENDING / ACTIVE / SUSPENDED / EXITED）
- `OrderStatus`（CREATED / COOLING_OFF / ACTIVE / COMPLETED / REFUNDED / TERMINATED）
- `InstallmentStatus`（PENDING / DELIVERED / PAID / OVERDUE / WRITTEN_OFF）
- `DeductionStatus`（PENDING / SUCCESS / FAILED）

## 六、关键业务约束（开发硬边界）

1. **资金不沉淀**：学员每期款项直达机构收单账户，平台不持有余额。
2. **先交付后代扣**：`InstallmentItem.contentDeliveredAt` 非空后方可发起代扣，违反则拒绝。
3. **未成年硬拦截**：实名核验年龄 < 18 立即阻断注册与下单。
4. **单级提成**：业务员只对直接促成订单计提，系统架构不支持多级分佣。
5. **合同主体约束**：`Order.sellerOrgId` 必填，合同甲方 = 机构，发票方 = 机构。
6. **7 天冷静期**：订单进入 `COOLING_OFF` 状态，期间学员可无理由退款。
