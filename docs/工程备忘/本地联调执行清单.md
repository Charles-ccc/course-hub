# 本地联调执行清单

## 1. 适用范围

本清单用于当前仓库的本地真实 API 联调，覆盖以下对象：

1. apps/api
2. apps/web-admin
3. apps/web-institution

目标是保证本地联调使用真实 MySQL 数据、真实 JWT + refresh token 链路，而不是仅依赖前端 mock。

## 2. 前置条件

1. Node.js >= 18
2. pnpm >= 8
3. Docker Desktop 已启动
4. 本地 MySQL 容器已启动

推荐先在仓库根目录执行：

```bash
docker compose up -d mysql
```

## 3. API 推荐启动方式

当前 apps/api 已修复开发态启动链路，本地真实联调优先直接使用 `pnpm dev`：

```bash
cd apps/api
DATABASE_URL='mysql://wangke:wangke123@127.0.0.1:3306/wangke' pnpm dev
```

当前 `pnpm dev` 的实际实现为：

1. 先执行一次 `pnpm build`
2. 再并行运行 `tsc --watch`
3. 同时用 `node --watch dist/main.js` 启动 Nest

这样做的原因是：

1. 之前的 `tsx watch src/main.ts` 路径不会发出 Nest 依赖注入所需的 decorator metadata。
2. 异常表现为控制器中的 service 在运行时变成 `undefined`，真实请求会返回 500。
3. 改成 `tsc watch + node --watch dist` 后，`pnpm dev` 下的真实登录请求已验证通过。

如需单次启动编译产物，也仍可使用：

```bash
cd apps/api
DATABASE_URL='mysql://wangke:wangke123@127.0.0.1:3306/wangke' pnpm build
DATABASE_URL='mysql://wangke:wangke123@127.0.0.1:3306/wangke' node dist/main.js
```

## 4. 数据库初始化与回基线

首次落表：

```bash
cd apps/api
DATABASE_URL='mysql://wangke:wangke123@127.0.0.1:3306/wangke' pnpm db:push
DATABASE_URL='mysql://wangke:wangke123@127.0.0.1:3306/wangke' pnpm db:seed
```

写接口联调后，如需恢复到仓库默认演示数据：

```bash
cd /Users/lc/Downloads/wangke-supermarket
DATABASE_URL='mysql://wangke:wangke123@127.0.0.1:3306/wangke' pnpm --filter api db:seed
```

## 5. Web 端本地联调方式

两个 web 端默认会请求线上地址 `https://api.happymaa.cn/api/v1`。

要改为本机 API，启动时显式注入环境变量：

### 5.1 web-admin

```bash
cd apps/web-admin
VITE_API_BASE_URL='http://127.0.0.1:3000/api/v1' pnpm dev
```

### 5.2 web-institution

```bash
cd apps/web-institution
VITE_API_BASE_URL='http://127.0.0.1:3000/api/v1' pnpm dev
```

说明：

1. 两个 web 端只有在显式打开前端 mock 开关时才会退回 mock。
2. 当前真实联调建议保持默认 mock 开关关闭。
3. 两个 web 端都已经接入 refresh token 会话持久化和 401 自动刷新。
