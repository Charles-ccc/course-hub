# 商用部署与资源备忘

## 1. 当前已确认资源

1. 云服务器：腾讯云轻量应用服务器（作为统一部署节点）。
2. 主域名：happymaa.cn。
3. 对象存储：腾讯云 COS，静态资源域名 static.happymaa.cn。
4. 规划中能力：短信验证码服务（用于注册、登录、找回密码、风控校验）。

## 2. 部署目标范围

本项目未来部署对象：

1. web-admin（平台运营后台）
2. web-institution（机构管理门户）
3. h5-staff（业务员工作台）
4. mp-student（学员小程序，服务端能力与统一后端同部署体系）

说明：两 web 端与 h5 端会部署在同一服务器体系中，小程序前端由平台托管，后端 API 与静态资源依赖同一云资源体系。

## 3. 建议域名与路由规划

1. 管理后台：admin.happymaa.cn
2. 机构门户：institution.happymaa.cn
3. 业务员 H5：staff.happymaa.cn
4. API 网关：api.happymaa.cn
5. 静态资源：static.happymaa.cn（已确定）

Nginx 建议：

- 前端静态站点走 CDN + Nginx 回源。
- API 统一反向代理至 NestJS 服务。
- TLS 统一由 Nginx 终止，内网走 HTTP。

## 4. 商用上线前必须补齐

1. DNS 与证书

- 完成所有子域名解析。
- 配置 TLS 证书自动续签。

2. 运行环境

- Node.js LTS、MySQL、Redis 版本锁定。
- 环境变量分层：dev、staging、prod。

3. 安全

- API 限流、登录防爆破、验证码防刷。
- 管理端强制 RBAC 权限校验。
- 审计日志与关键操作留痕（审批、禁用、结算、配置变更）。

4. 可观测

- 日志聚合（JSON 日志 + 检索）。
- 指标监控（CPU、内存、磁盘、QPS、错误率、P95）。
- 告警策略（短信服务故障、签约服务故障、支付通道故障）。

5. 数据可靠性

- MySQL 定时备份与恢复演练。
- Redis 持久化与重启恢复验证。
- 结算与账务任务幂等、防重、失败补偿。

## 5. COS 使用规范

1. 资源分类目录建议

- /public：公开素材（课程封面、说明图）
- /private：需签名访问资源（如敏感导出）
- /temp：临时文件（定时清理）

2. 上传规范

- 文件命名使用 UUID，避免覆盖。
- 写入 metadata：上传人、业务模块、上传时间。
- 限制 mime type 与文件大小。

3. 访问控制

- 公共资源走 CDN 缓存。
- 私有资源使用短时签名 URL。

## 6. 短信验证码服务接入备忘

1. 接口抽象

- SmsProvider.sendCode(phone, scene, templateId, params)
- SmsProvider.verifyCode(phone, scene, code)

2. 场景枚举

- REGISTER
- LOGIN
- RESET_PASSWORD
- HIGH_RISK_OPERATION

3. 商用要求

- 同手机号发送频率限制（如 60 秒）。
- 同 IP 限流与黑名单策略。
- 验证码错误次数上限与冷却时间。
- 服务商故障降级策略（主备通道或排队重试）。
