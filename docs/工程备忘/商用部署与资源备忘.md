## 4. 商用上线前必须补齐

### 4.0 首次部署数据库初始化 ⚠️

> **重要**：新部署或数据库重建时必须执行以下步骤，否则 API 登录报错"表不存在"。

在生产服务器执行（在 `/www/wwwroot/wangke-supermarket/apps/api` 目录）：

```bash
# 1. 创建基础表结构
pnpm db:push

# 2. 重命名表和字段（Institution → Insitution）
pnpm db:rename:insitution

# 3. 添加 Salesman 表扩展字段
pnpm db:add:staff

# 4. 创建审计日志表
pnpm db:add:audit
```

> 说明：若使用 docker-compose 部署，在容器启动后立即执行上述命令。

1. DNS 与证书

- 完成所有子域名解析。
- 配置 TLS 证书自动续签。

2. 运行环境

- Node.js LTS、MySQL、Redis 版本锁定。
- 环境变量分层：dev、staging、prod。

3. 安全

- API 限流、登录防爆破、验证码防刷。

## 5. COS 使用规范

1. 资源分类目录建议

- /public：公开素材（课程封面、说明图）
- /private：需签名访问资源（如敏感导出）
- /temp：临时文件（定时清理）

2. 上传规范

- 文件命名使用 UUID，避免覆盖。
- 写入 metadata：上传人、业务模块、上传时间。
- 限制 mime type 与文件大小。

3. 访问控制

- 公共资源走 CDN 缓存。
- 私有资源使用短时签名 URL。

## 6. 短信验证码服务接入备忘

1. 接口抽象

- SmsProvider.sendCode(phone, scene, templateId, params)
- SmsProvider.verifyCode(phone, scene, code)

2. 场景枚举

- REGISTER
- LOGIN
- RESET_PASSWORD
- HIGH_RISK_OPERATION

3. 商用要求

- 同手机号发送频率限制（如 60 秒）。
- 同 IP 限流与黑名单策略。
- 验证码错误次数上限与冷却时间。
- 服务商故障降级策略（主备通道或排队重试）。

## 7. 统一仓库部署方案（2026-06-11 更新）

当前所有代码（API + web-admin + web-insitution）已集中到服务器 `/www/wwwroot/wangke-supermarket` 仓库，采用统一部署策略。

### 7.1 快速部署清单（服务器内执行）

```bash
# 登录服务器
ssh root@124.220.78.20
cd /www/wwwroot/wangke-supermarket

# 更新代码
git pull origin main

# 确保 Redis 在线（限流与鉴权依赖）
docker compose up -d redis
redis-cli -h 127.0.0.1 -p 6379 ping

# 安装依赖
pnpm install --frozen-lockfile

# 数据库更新（如有 schema 变更）
pnpm --filter api db:push

# 构建三端应用
pnpm --filter api build
pnpm --filter web-admin build
pnpm --filter web-insitution build

# 同步前端资源
rsync -av --delete apps/web-admin/dist/ /www/wwwroot/admin/
rsync -av --delete apps/web-insitution/dist/ /www/wwwroot/insitution/

# 重启 API 服务
pm2 restart wangke-api
pm2 logs wangke-api --lines 50

# 验证 Nginx 并重载
nginx -t && systemctl reload nginx

# 健康检查
curl http://127.0.0.1:3000/health
```

### 7.3 REDIS_URL 配置规范（新增）

后端限流模块已改为“启动强依赖 Redis”，`REDIS_URL` 缺失时 API 将拒绝启动。

1. 推荐配置值（同机部署）

```bash
REDIS_URL=redis://127.0.0.1:6379
```

2. 若 Redis 设置密码

```bash
REDIS_URL=redis://:YourStrongPassword@127.0.0.1:6379/0
```

3. PM2 环境变量示例（推荐）

```bash
cd /www/wwwroot/wangke-supermarket/apps/api

DATABASE_URL="mysql://wangke:wangke123@127.0.0.1:3306/wangke" \
REDIS_URL="redis://127.0.0.1:6379" \
JWT_ACCESS_SECRET="<replace-with-prod-secret>" \
JWT_REFRESH_SECRET="<replace-with-prod-secret>" \
pm2 start dist/main.js --name wangke-api --update-env

pm2 save
```

4. 启动后验证

```bash
# Redis 连接验证
redis-cli -h 127.0.0.1 -p 6379 ping

# API 进程日志检查（应包含 Redis healthy 日志）
pm2 logs wangke-api --lines 100
```

5. 常见故障排查

- 报错 `REDIS_URL is required for rate limiting`：未注入环境变量或 PM2 未 `--update-env`。
- 报错 `ECONNREFUSED 127.0.0.1:6379`：Redis 未启动或端口未开放。
- 报错鉴权限流异常：优先检查 `redis-cli ping` 与 API 日志中 Redis 初始化状态。

### 7.2 部署核心优势

- 无需本机打包与 scp 上传，减少网络传输开销。
- 每次发布仅需 git pull + 构建 + 重启，流程简化且可审计。
- 支持一键回滚：`git reset --hard <commit-id>` + 重新构建。
- 所有环境变量与敏感配置由服务器本地管理，不经过 Git。
