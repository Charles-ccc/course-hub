## 二、已完整实现且与设计文档一致的部分 ✅

### 2.1 后端业务核心模块（NestJS）

- **用户/实名**（[apps/api/src/modules/user](apps/api/src/modules/user)）
  - 年龄 < 18 硬拦截（错误码 40001）
  - 身份证号哈希 + 加密双字段存储
- **订单**（[apps/api/src/modules/order](apps/api/src/modules/order)）
  - 卖方 = 机构强制（`order.sellerOrgId` 必填）
  - 客单价上限校验（10000 元）
  - 7 天冷静期、冷静期内全额无理由退款
  - 合同甲方 = 机构
- **分期与代扣**（[apps/api/src/modules/installment](apps/api/src/modules/installment)）
  - **先交付后扣款**：未置位 `contentDeliveredAt` 直接拒绝（错误码 42201）
  - 收款方恒等于机构
  - 多通道优先级（微信支付分 → 芝麻 → 银行卡）架构就位
- **信用风控**（[apps/api/src/modules/credit](apps/api/src/modules/credit)）
  - 商户主体校验（必须是机构）
  - 不存储原始信用分（无 `rawScore` 字段）
  - 内置 Provider 降级策略
- **提成与拉新**（[apps/api/src/modules/commission](apps/api/src/modules/commission)、[apps/api/src/modules/referral](apps/api/src/modules/referral)）
  - 两段式提成（成单 3% + 履约 1%）
  - 逾期 held / 扣回
  - 拉新单级 50 元奖励 + 20% 个税代扣
  - 同一被邀请人只奖励一次（防刷）
- **学习与打卡**（[apps/api/src/modules/learning](apps/api/src/modules/learning)）
  - 内容交付解锁置位 `contentDeliveredAt`
  - 学习激励发放来源 = 机构
- **结算**（[apps/api/src/modules/settlement](apps/api/src/modules/settlement)）
  - 月结自动生成、GMV、服务费按 `settlementFeeRate` 计算

### 2.2 Prisma 数据模型

### 3.2 代扣重试机制 ❌

- ✅ 日常代扣定时任务 `@Cron(EVERY_DAY_AT_9AM)` 存在
- ❌ **缺失** D+1 / D+3 / D+7 三档重试调度
- ❌ 重试次数上限与最终转逾期的状态编排不完整

### 3.3 提成/拉新边界

- ❌ 业务员自购（self-buy）不计提成的拦截缺失
- ❌ 多级分佣 / 多级拉新的显式防护缺失（虽然 schema 不支持，但缺审计性拒绝）

## 八、优先修复建议（按重要性）

### 🔴 Critical（阻断生产）

1. **代扣重试调度**：补 D+1 / D+3 / D+7 定时任务，重试失败终态自动转逾期
2. **支付/代扣真实接入**：实现 `callPaymentChannel()`（易宝或鸿蜂，参考 [docs/鸿蜂代扣接口设计文档V1.5.docx](鸿蜂代扣接口设计文档V1.5.docx)）
3. **实名核验真实接入**：公安部四要素

### 🟠 High（业务完整性）

4. **VOD 集成**：阿里云/腾讯 VOD 完整链路 + 与代扣的内容交付一一对应
5. **电子合同真实接入**：替换 mock URL，打通人脸核身
6. **合规监控**：催收时段/频次/话术系统强制 + 审计日志

### 🟡 Medium（数据完整性）

7. **财务对账**：与支付方自动对账 + 差异工单
8. **税务/发票**：提成 > 500 元代开票
9. **Staff self-buy 防护**：`commission.service` 中拒绝 `staffId === order.attributionStaffId`
10. **通知系统真实接入**：阿里云短信 + 模板消息
