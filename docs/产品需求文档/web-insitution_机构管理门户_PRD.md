# web-insitution · 机构管理门户 PRD

---

# 1. 背景

机构管理门户是供合作教育机构的运营人员日常使用的管理工具。在「欢乐学」的业务模型中，机构是合同甲方、课程销售方和资金收款方，承担课程交付和学员服务的核心责任。

机构在本平台的日常工作包括：发布课程商品、追踪学员履约状态、处理逾期坏账、回复学员答疑，以及与平台对账每月结算款项。

## 1.1 功能范围

- 建立课程生命周期管理：支持机构创建课程并控制上下架状态
- 订单与分期透明化：机构可实时查看学员订单状态及每期还款进度
- 提供合规的逾期处置通道：坏账核销是平台允许的唯一逾期处置手段
- 落地学员服务闭环：机构可在线回复学员提问

## 1.2 价值

对机构运营人员：核心操作在一个界面内完成，减少与平台来回沟通的成本。  
对平台合规：将机构行为（尤其是逾期处置）限制在系统内，杜绝线下催收风险。

# 3. 需求

## 3.1 登录

作为机构管理员，我希望用手机号和密码登录门户，以便进入我的机构管理工作台。

验收条件：

1. 登录表单包含「手机号」和「密码」两个必填项。
2. 点击「登录」后按钮进入 loading 状态并禁用，防止重复提交。
3. 登录成功后，将 token、orgId、orgName 持久化到 localStorage，并跳转至 `/dashboard`。
4. 登录失败时，toast 提示：手机号或密码错误。
5. 未登录状态下访问任意页面，自动跳转至 `/login`。
6. 任意接口返回 HTTP 401 时，清空本地 token 并跳转 `/login`；dev token（以 `dev-` 开头）不触发此逻辑。
7. 左侧导航「退出登录」清空本地存储后跳转至 `/login`。

## 3.2 运营概览

作为机构管理员，我希望进入门户后快速了解当前经营状态，以便判断是否有需要处理的事项。

验收条件：

**指标卡片**

1. 页面展示三个指标卡片：本月订单数（全状态订单总条数）、学习中学员数（订单状态为 ACTIVE 的数量）、累计 GMV（元，所有订单 totalAmount 之和换算）。
2. 逾期率指标卡片保留占位，显示「--」，暂不实现数据计算。
3. 指标数据从订单列表本地计算得出，无单独的统计接口。

**最近订单预览** 4. 页面下方展示最近 10 条订单，列包括：学员（脱敏姓名格式：姓氏 + _，如「林_」）、课程名称、金额（元）、状态、下单时间。5. 所有学员姓名在界面层统一做脱敏处理，不展示完整真实姓名。

**数据加载** 6. 页面挂载时自动请求订单列表和机构资料，加载期间显示骨架屏。

## 3.3 课程管理

### 3.3.1 课程列表

作为机构管理员，我希望查看自己发布的所有课程，以便了解上下架状态并进行管理操作。

验收条件：

1. 列表展示字段：课程名称、讲师信息（摘要）、课程总价（元）、分期期数、每期金额（元）、状态、操作。
2. 操作列规则：OFFLINE 状态显示「上架」按钮；ONLINE 状态显示「下架」按钮。
3. 数据只加载本机构的课程（后端通过 orgId 过滤），不显示其他机构的课程。
4. 列表最多展示 100 条，当前无分页。

---

### 3.3.2 创建课程

作为机构管理员，我希望在门户内发布新课程，以便学员可以在小程序中发现并购买。

验收条件：

**表单字段与校验**

1. 创建课程入口为列表页右上角「新建课程」按钮，点击弹出 Modal 表单。
2. 表单字段：课程名称（必填，最长 100 字）、课程简介（必填，最长 2000 字）、讲师信息（必填，最长 500 字）、课程总价（必填，正整数，单位「元」）、分期期数（必填，1～36 的整数）。
3. 任意必填项为空时，点击「确认创建」后高亮对应字段并阻止提交。

**提交与结果** 4. 提交时，前端将「课程总价（元）」乘以 100 转换为分后发送。5. 创建成功后，Modal 关闭，课程列表刷新，新课程出现在列表中。6. 新建课程默认状态为 OFFLINE（下架），需手动上架后学员方可见。7. 若课程总价超过平台价格上限，接口返回错误码 40002，Modal 内展示错误提示：课程定价超出平台允许的最高金额，请调低定价后重试。

---

### 3.3.3 课程上架 / 下架

作为机构管理员，我希望控制课程的上下架状态，以便决定哪些课程对学员可见。

验收条件：

1. 点击「上架」或「下架」按钮，弹出二次确认对话框。
2. 上架确认文案：上架后学员将可以在小程序中看到并购买此课程，是否继续？
3. 下架确认文案：下架后学员将无法发现和购买此课程，已有进行中的订单不受影响，是否继续？
4. 确认后发送请求，成功则列表刷新，课程状态即时更新。
5. 课程下架后，学员小程序端立即看不到该课程；已签约的订单继续正常履约，不受影响。
6. 课程不支持删除操作，只能下架。

## 3.4 订单管理

### 3.4.1 订单列表

作为机构管理员，我希望按状态筛选查看学员订单，以便了解各阶段的履约情况。

验收条件：

1. 页面顶部提供订单状态筛选 Tab：全部 / 待签约 / 冷静期中 / 学习中 / 已完成 / 已退款 / 已终止。
2. 各 Tab 对应的订单状态：待签约 = CREATED，冷静期中 = COOLING_OFF，学习中 = ACTIVE，已完成 = COMPLETED，已退款 = REFUNDED，已终止 = TERMINATED。
3. 列表展示字段：订单号、学员（脱敏，如「林\*」）、课程名称、订单金额（元）、分期期数、每期金额（元）、状态、冷静期截止（仅 COOLING_OFF 状态显示）、下单时间。
4. 所有学员姓名统一做脱敏处理，格式为「姓氏 + \*」。
5. 机构对订单无主动操作权，退款和终止由学员或平台发起，列表仅供查看。

---

### 3.4.2 分期明细

作为机构管理员，我希望查看订单的分期还款明细，以便了解每期的收款和内容交付状态。

验收条件：

1. 点击订单行可展开查看该订单的分期明细列表。
2. 分期明细字段：期次（第 X 期）、到期日、计划金额（元）、已扣金额（元）、内容交付时间、期次状态。
3. 期次状态展示规则：
   - PENDING → 待扣款
   - DELIVERED → 已交付，待代扣
   - PAID → 已还款
   - OVERDUE → 已逾期（红色标识）
   - WRITTEN_OFF → 已核销（灰色标识）
4. 内容交付时间为空时，显示「-」，表示本期内容尚未交付，平台不会对该期发起代扣。
5. 内容交付是平台代扣的前提，机构须知晓。

## 3.5 逾期管理

作为机构管理员，我希望查看所有逾期期次并对无法追回的款项执行核销，以便清理账期。

> **合规声明（页面顶部固定展示）：** 平台明确禁止任何形式的催收骚扰行为。逾期处置的唯一合法手段是在本页面执行坏账核销。如发现线下催收行为，平台将立即暂停合作资格。

验收条件：

**逾期列表**

1. 列表展示所有状态为 OVERDUE 的分期期次，字段：订单号、学员（脱敏）、课程名称、第 X 期、应还日期、逾期天数（今日与应还日期之差，前端计算）、计划金额（元）。
2. 合规声明文字在页面顶部以醒目样式固定展示，不可关闭。

**坏账核销** 3. 点击「坏账核销」按钮，弹出确认对话框。4. 对话框内展示：学员（脱敏）、课程名称、第 X 期、计划金额（元）。5. 对话框提供「核销备注」文本域，必填，用于记录核销原因（如：学员失联、无力偿还等）。6. 对话框包含不可撤销提示：核销操作不可撤销。核销后该期款项视为机构自行核销，平台不参与追偿。7. 「确认核销」按钮为橙色警告样式。8. 请求成功后，该期次状态变为 WRITTEN_OFF，从逾期列表中移除。9. 核销后，同一订单的其他未逾期期次不受影响，继续正常履约。

---

## 3.6 结算对账

作为机构管理员，我希望查看平台对我的保证金余额和月度结算记录，以便核对应得款项是否到账。

验收条件：

**保证金展示**

1. 页面顶部展示当前机构的履约保证金余额（元），数据来源于 `/v1/org/deposit` 接口。
2. 页面展示本机构的结算费率（%），帮助机构了解平台服务费计算依据。

**结算计算说明** 3. 页面以提示文字说明结算逻辑：机构本月实得 = 本月 GMV × (1 - 结算费率%)。

**月度结算记录** 4. 以表格展示所有月度结算记录，字段：结算周期（YYYY-MM）、本期 GMV（元）、平台服务费（元）、机构实得（元，前端计算）、结算状态。5. 机构实得 = 本期 GMV − 平台服务费，前端本地计算展示。6. 结算状态展示规则：PENDING → 待结算；SETTLED → 已结算。7. 机构无法在此页面发起结算，结算操作由平台运营人员在 web-admin 执行。

---

## 3.7 答疑管理

作为机构管理员，我希望及时回复学员提出的课程问题，以便提升学员学习体验并降低退款率。

验收条件：

**问题列表**

1. 页面顶部提供筛选 Tab：待回复 / 全部。
2. 「待回复」Tab 只展示 replied=false 的问题；「全部」展示所有问题。
3. 列表字段：学员（脱敏，如「林\*」）、提问内容、提问时间、回复状态（待回复/已回复 Badge）、回复内容（已回复时展示）、回复时间（已回复时展示）。
4. 状态为「待回复」的问题行，操作列显示「回复」按钮。
5. 状态为「已回复」的问题行，操作列为空。

**回复问题** 6. 点击「回复」后，在问题行下方展开输入区域，包含文本域和「提交回复」按钮。7. 回复内容为必填，空内容点击「提交回复」时不发送请求。8. 提交成功后，该问题状态变为「已回复」，回复内容和时间展示在列表行中，输入区域收起。9. 学员可在小程序端看到机构的回复内容。10. 回复提交后不可修改，当前版本无编辑接口。
