# web-admin · 平台运营后台 PRD

---

# 1. 背景

「欢乐学」是一个以先学后付为核心的在线课程销售平台。平台不垫资、不归集资金，合同甲方与收款方均为合作机构，平台扮演信用撮合与技术服务角色。

运营后台是平台自有运营人员使用的**内部管理控制台**，不对外公开。运营人员通过该系统完成机构准入审核、业务员状态管控、平台健康度监控、月度结算执行和系统参数调整等核心工作。

## 1.1 功能范围

- 建立机构审批流程：收集费率参数，控制机构从注册到上线的准入门槛
- 提供平台健康度实时看板：聚合还款率、逾期率、退款率等关键指标，异常时主动预警
- 落地结算管理闭环：平台按月执行结算操作，标记服务费到账状态
- 支持系统级参数动态配置：价格上限和支付通道开关无需发版即可生效

## 1.2 价值

对运营人员：减少需要进数据库排查的场景，核心管理动作在页面内完成。  
对平台：将机构审核、结算操作结构化，沉淀可追溯的操作记录。

# 3. 需求

## 3.1 登录

作为平台运营人员，我希望通过用户名和密码登录后台，以便进入各管理模块执行操作。

验收条件：

1. 登录表单包含「用户名」和「密码」两个输入项，均为必填。
2. 点击「登录」后，按钮进入 loading 状态并禁用，防止重复提交。
3. 登录成功后，将 token 和角色信息持久化到 localStorage，并自动跳转至运营概览页。
4. 登录失败时，toast 提示：用户名或密码错误。
5. 已登录状态下访问 `/login`，自动跳转至 `/dashboard`。
6. 未登录状态下访问任意页面，自动跳转至 `/login`。
7. 任意接口返回 HTTP 401 时，清空本地 token 并跳转至 `/login`；dev token（以 `dev-` 开头）不触发此逻辑。
8. 顶部导航「退出登录」操作清空 localStorage 中的 token 后跳转至 `/login`。

---

## 3.2 运营概览

作为平台运营人员，我希望在一个页面内看到平台当前健康状态，以便快速判断是否有需要介入的异常。

验收条件：

**指标展示**

1. 页面展示以下 5 个指标卡片：平台总 GMV（元）、活跃订单数、还款率（%）、逾期率（%）、退款率（%）。
2. 金额类字段统一由「分」换算为「元」后展示，保留两位小数。

**预警规则** 3. 还款率 < 90% 时，对应指标卡片显示橙色警告样式。4. 逾期率 > 10% 时，对应指标卡片显示橙色警告样式。5. 退款率 > 5% 时，对应指标卡片显示橙色警告样式。6. 所有指标均正常时，页面顶部显示绿色状态提示：平台运营状态健康。7. 任意指标触发预警时，页面顶部显示橙色状态提示：存在需关注的指标。

**数据加载** 8. 页面挂载时自动请求健康度数据，加载期间显示骨架屏。9. 无自动刷新，刷新页面可获取最新数据。

## 3.3 机构管理

### 3.3.1 机构列表

作为平台运营人员，我希望查看所有合作机构并按状态筛选，以便快速定位需要操作的机构。

验收条件：

1. 列表展示字段：机构名称、统一社会信用代码、结算费率、保证金余额（元）、累计 GMV（元）、平台累计服务费（元）、状态、操作。
2. 页面顶部提供状态筛选 Tab：全部 / 待审核 / 运营中 / 已暂停 / 已退出。
3. 切换 Tab 后列表重新请求数据，不做前端本地过滤。
4. 操作列根据机构状态动态显示：
   - PENDING → 显示「审核通过」按钮
   - ACTIVE → 显示「暂停」按钮
   - SUSPENDED / EXITED → 操作列为空

---

### 3.3.2 审核通过机构

作为平台运营人员，我希望在审核机构时设定结算费率，以便机构激活后按约定比例收取服务费。

验收条件：

1. 点击 PENDING 机构行的「审核通过」，弹出确认对话框。
2. 对话框中展示机构名称和统一社会信用代码，供运营确认身份。
3. 对话框提供「结算费率（%）」输入项，必填，取值范围 0.00 ~ 100.00，精度两位小数。
4. 对话框包含提示文字：费率设定后无法通过界面修改，请谨慎填写。
5. 点击「取消」关闭对话框，不触发任何操作。
6. 点击「确认通过」后，按钮进入 loading 状态并禁用。
7. 请求成功后，对话框关闭，列表刷新，该机构状态变为 ACTIVE。
8. 请求失败时，对话框内展示错误提示，不关闭对话框。
9. 审核通过后，机构管理员可在 web-insitution 正常登录并发布课程。

---

### 3.3.3 暂停机构

作为平台运营人员，我希望对违规或异常的机构执行暂停操作，以便阻止新学员继续购课。

验收条件：

1. 点击 ACTIVE 机构行的「暂停」，弹出确认对话框。
2. 对话框中展示机构名称，并提供「暂停原因」文本域，必填。
3. 对话框包含业务影响说明：暂停后该机构课程将对学员不可见，无法新建订单；已有进行中的订单不受影响。
4. 「确认暂停」按钮为红色警告样式，需二次确认。
5. 请求成功后，机构状态变为 SUSPENDED，列表刷新。
6. 请求失败时，对话框内展示错误提示，不关闭对话框。
7. 机构暂停后，该机构业务员的推广二维码仍可进入，但学员下单时报错（由后端拦截）。

---

### 3.3.4 新增机构

作为平台运营人员，我希望手动创建机构记录，以便为无自主注册能力的机构启用平台功能。

验收条件：

1. 机构列表页面提供「新增机构」按钮。
2. 点击按钮后弹出对话框，包含以下输入项：
   - 机构名称（必填，最长 100 字符）
   - 统一社会信用代码（必填，固定 18 位）
   - 保证金（非必填，金额单位元，默认 0）
3. 对话框包含表单验证提示。
4. 点击「确认」后，按钮进入 loading 状态并禁用。
5. 请求成功后，对话框关闭，列表刷新，新增机构显示在列表顶部，初始状态为 PENDING。
6. 请求失败时，对话框内展示错误提示，不关闭对话框。

---

### 3.3.5 编辑机构

作为平台运营人员，我希望修改机构的基本信息，以便及时更新错误或过期的信息。

验收条件：

1. 机构列表每行右侧提供「编辑」按钮，仅 PENDING 和 ACTIVE 状态可编辑。
2. 点击按钮后弹出对话框，展示该机构的当前信息，可编辑字段为：
   - 机构名称
   - 统一社会信用代码
   - 保证金
3. 已发生结算的机构，保证金字段不可修改。
4. 点击「保存」后，按钮进入 loading 状态并禁用。
5. 请求成功后，对话框关闭，列表刷新，新数据即时显示。
6. 请求失败时，对话框内展示错误提示，不关闭对话框。

---

### 3.3.6 删除机构

作为平台运营人员，我希望删除错误或无效的机构记录，以便保持数据清洁。

验收条件：

1. 机构列表每行右侧提供「删除」按钮，仅 PENDING 状态可删除。
2. 点击按钮后弹出二次确认对话框，展示机构名称。
3. 对话框包含警告文字：删除后该机构的所有关联数据（如草稿课程）将被删除，且无法恢复。
4. 「确认删除」按钮为红色警告样式。
5. 请求成功后，对话框关闭，列表刷新，该机构行消失。
6. 请求失败时，对话框内展示错误提示，不关闭对话框。

## 3.6 数据报表

### 3.6.1 GMV 月度报表

1. 页面顶部提供月份选择器，默认选中当前月份，格式 YYYY-MM。
2. 切换月份后自动触发数据请求，加载期间显示 loading 状态。

5. 无数据时展示空状态提示：暂无数据。

---

### 3.6.2 逾期率监控

## 3.7 结算管理

作为平台运营人员，我希望对每月的机构收入执行结算确认，以便完成平台与机构之间的财务对账。

结算计算规则：机构本月实得 = 本月 GMV − 平台服务费；平台服务费 = 本月 GMV × 该机构结算费率。

验收条件：

**列表**

1. 列表展示字段：机构名称、结算周期（YYYY-MM）、本期 GMV（元）、平台服务费（元）、机构实得（元，前端计算）、结算状态、操作。
2. 机构实得 = GMV 减去平台服务费，前端本地计算展示，不从接口取值。
3. 结算状态为 PENDING 时，操作列显示「执行结算」按钮；SETTLED 时操作列为空。

**执行结算** 4. 点击「执行结算」，弹出确认对话框，对话框内展示：机构名称、结算周期、GMV、平台服务费、机构实得金额。5. 对话框包含提示文字：结算操作不可撤销，请确认金额无误后继续。6. 请求成功后，记录状态变为 SETTLED，列表刷新。7. 机构在 web-insitution 结算对账页面可同步看到该条记录状态变为「已结算」。

## 3.8 系统配置

### 3.8.1 价格上限配置

作为平台运营人员，我希望动态调整学员所有订单的未还款总金额上限，以便控制信用风险敞口。

验收条件：

1. 页面展示当前生效的价格上限（元）。
2. 提供数字输入框，允许输入新的价格上限，单位为「元」，必须为正整数。
3. 保存成功后，toast 提示：配置已更新，即时生效。
4. 配置生效后，学员下单时若课程价格超出上限，后端返回错误码 40002，此时需提示学员该课程暂不支持分期购买。
5. 价格上限修改立即生效，不影响已有订单。

---

### 3.8.2 芝麻先享开关

作为平台运营人员，我希望控制芝麻先享支付通道的启用状态，以便在商户资质就绪前避免学员支付失败。

验收条件：

1. 页面展示「芝麻先享」开关当前状态（开启/关闭）。
2. 开关旁显示提示文字：开启前需确认已完成支付宝商户主体入驻和法务审核，否则将导致学员支付失败。
3. 切换开关后需点击「保存」才生效，切换开关本身不触发请求。
4. 保存成功后，toast 提示：配置已更新，即时生效。
5. 开关关闭时，学员端芝麻先享支付选项不可用，后端返回错误码 42202（CREDIT_PROVIDER_NOT_ENABLED）。
