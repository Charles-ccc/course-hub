# mp-student · 学员小程序 PRD

---

# 1. 背景

学员小程序是「欢乐学」面向学员的核心消费端，同时支持微信小程序和支付宝小程序两个平台（基于 Taro 3 框架开发）。

学员通过业务员的邀请链接或二维码进入小程序，完成注册、实名认证、浏览课程、下单签约、分期还款和在线学习的完整消费旅程。「先学后付」是本产品最核心的商业模式：学员先学习本期课程，课程结束后系统自动代扣学费，降低学员的决策门槛。

## 1.1 功能范围

- 归因追踪：在应用启动时捕获来源（业务员/学员邀请），写入归因数据
- 注册与认证：手机号注册、实名认证（含未成年人拦截）、找回密码
- 课程发现：关键词搜索、课程列表、课程详情和免费试学
- 下单与签约：免费试学、创建订单、法大大电子签约、
- 学习：视频课播放、联系老师、人脸打卡获取学习激励
- 订单管理：查看订单列表、学费分期支付进度
- 个人中心：查看个人信息、实名认证入口、退出登录

## 1.2 价值

对学员：先体验后付款，降低购课决策风险；电子签约保障权益透明。  
对平台：归因追踪保障业务员提成准确计算；实名认证和合同签署构建合规基础。

# 3. 需求

## 3.0 需求背景与业务目标

### 3.0.1 背景

为保障用户来源可控，避免无效注册，平台采用**业务员邀请制注册**模式。用户必须通过**机构码 / 业务员邀请码**完成注册，注册后支持多种登录方式。所有注册用户必须归属某一业务员或机构，作为后续业绩统计、提成计算的数据基础。

### 3.0.2 业务目标

- ✅ 所有注册用户必须绑定业务员，确保用户来源可控
- ✅ 防止非目标用户随意注册，降低无效流量
- ✅ 记录完整的邀请关系链，支撑业绩追踪与提成计算
- ✅ 降低登录门槛，支持多种登录方式，提升留存率
- ✅ 提升注册转化率，缩短注册链路

### 3.0.3 用户角色

| 角色       | 说明                                                 |
| ---------- | ---------------------------------------------------- |
| 普通学员   | 被邀请注册的小程序用户，通过业务员邀请码完成注册     |
| 业务员     | 拥有机构码和邀请码，负责邀请学员，后续与学员提成挂钩 |
| 后台管理员 | 管理业务员、机构码状态、邀请关系审核                 |

### 3.0.4 业务流程（文字版）

**【未注册学员】**

```
打开小程序 → [自动捕获 staffId / invCode]
→ 点击注册 → 输入机构码
→ 后端校验（存在性、启用状态、过期期限）
→ 展示机构信息 + 业务员名称
→ 输入手机号 → 获取短信验证码
→ 填写验证码 + 密码 → 完成注册
→ 自动绑定业务员 → 跳转实名认证页
```

**【已注册学员】**

```
打开小程序 → [检查本地 token]
→ token 有效：直接跳转首页
→ token 失效：展示引导页
→ 选择登录方式：
   ├─ 手机号 + 密码 (支持忘记密码)
   ├─ 手机号 + 验证码 (免密登录)
   ├─ 微信授权登录
   └─ 支付宝授权登录
→ 首次授权：弹出手机号绑定流程
→ 已绑定：直接登录
```

## 3.1 应用启动与归因

作为平台，我希望在学员首次进入小程序时自动捕获来源归因，以便准确记录学员与业务员的绑定关系和学员互邀关系。

验收条件：

1. 应用启动时（onLaunch），解析启动参数中的 `?staff=<staffId>` 字段，若存在则写入本地存储键 `referrerStaffId`。
2. 启动参数中含 `?inv=<studentId>` 时，写入本地存储键 `referrerStudentId`。
3. 归因数据在后续注册请求时一并传入，其中 `referrerStaffId` 透传为 `inviteSalesmanId`；后端仅在首次注册成功时写入绑定关系，已绑定关系不可被后续请求覆盖。
4. 若已登录（token 有效），跳过引导页直接跳转至首页 Tab。
5. 若本地存储中存在 `registeredPhone`，尝试通过 Taro.login() 获取平台 authCode 并发起静默登录（`POST /api/v1/auth/silent-login`），成功后跳转首页，失败后展示引导页。
6. 引导页提供「立即注册」和「已有账号，去登录」两个入口按钮。

## 3.2 注册功能

### 3.2.1 机构码校验

作为学员，我希望通过输入机构码完成身份验证，以便确保我被正确的机构和业务员邀请。

验收条件：

**校验流程**

1. 注册表单首项为「机构码」输入框，用户输入机构码后，实时请求 `POST /api/v1/auth/org-code/validate { orgCode }`。
2. 校验项包括：
   - 机构码是否存在
   - 是否启用（status = ACTIVE）
   - 是否过期（expireAt > now）
3. 校验成功后，页面展示确认卡片：「您将加入：XX 机构（业务员：张三）」，展示机构名称和关联业务员信息。
4. 校验失败时，根据错误码展示对应提示（见 3.7 异常处理）。

**限流与防暴力**

5. 单手机号在 24 小时内机构码校验失败 ≥ 5 次，冻结该手机号机构码输入框 30 分钟；冻结期间提示「尝试次数过多，请 30 分钟后重试」。
6. 冻结记录存储至后端，端上不得绕过限流。

---

### 3.2.2 手机号注册

作为学员，我希望通过手机号和短信验证码完成注册，以便开始使用平台购课。

验收条件：

**表单字段与校验**

1. 注册表单包含：机构码（必填）、手机号（必填）、短信验证码（必填）、密码（必填）、确认密码（必填）。
2. 机构码通过校验后，显示已验证状态，用户无法重新修改。
3. 点击「获取验证码」前，校验手机号是否为 11 位数字；不足 11 位时拦截并提示：**请输入正确的手机号**。
4. 点击「获取验证码」发送 `POST /api/v1/auth/sms/send { phone, type: "register" }`，成功后按钮进入 60 秒倒计时并禁用，倒计时结束后恢复为「获取验证码」。
5. 同一手机号 60 秒内不可重复发送验证码；24 小时内最多发送 5 次，超出后拦截并提示：**今日获取次数已达上限，请明天重试**。
6. 验证码有效期为 5 分钟；用户输入验证码时前端无需验证有效期（后端校验），但页面可展示倒计时提示。
7. 密码长度须 ≥ 6 位，不满足时提交拦截，提示：**密码长度不少于 6 位**。
8. 确认密码与密码不一致时提交拦截，提示：**两次密码输入不一致**。

**提交与结果**

9. 提交 `POST /api/v1/auth/phone/register { phone, code, password, orgCode, inviteSalesmanId }`，其中 `inviteSalesmanId` 来自本地存储的 `referrerStaffId`。
10. 注册成功后，后端自动写入邀请关系（user → inviteSalesmanId）；已绑定关系不可被后续请求覆盖。
11. 注册成功后将 token 和 profile 存储至 Zustand store，同时写入本地存储 `registeredPhone = phone`。
12. 同一手机号仅可注册一次；若手机号已注册，接口返回错误码 40003（PHONE_ALREADY_REGISTERED），页面提示：**该手机号已注册，请直接登录**，并提供「去登录」按钮。
13. 注册成功后跳转至实名认证页（`/pages/auth/realname`），实名认证完成后方可购课。
14. 注册失败时页面展示接口返回的错误信息。

## 3.3 登录功能

### 3.3.1 登录方式概览

作为学员，我希望使用多种方式登录，以便灵活选择最便捷的登录方法。

支持的登录方式：

| 登录方式        | 说明                   | 首次使用     | 适用场景             |
| --------------- | ---------------------- | ------------ | -------------------- |
| 手机号 + 密码   | 传统账号密码登录       | 直接登录     | 已绑定且记得密码     |
| 手机号 + 验证码 | 免密登录，仅需短信验证 | 直接登录     | 忘记密码或快速登录   |
| 微信授权        | 通过微信快速登录       | 需绑定手机号 | 微信小程序用户首选   |
| 支付宝授权      | 通过支付宝快速登录     | 需绑定手机号 | 支付宝小程序用户首选 |

### 3.3.2 手机号 + 密码登录

作为学员，我希望用手机号和密码登录，以便继续我的学习进程。

验收条件：

1. 登录表单包含「手机号」和「密码」两个必填项，均非空时才允许提交。
2. 登录前端无需预先校验密码格式，直接调用 `POST /api/v1/auth/phone/login { phone, password }`。
3. 登录成功后，保存 token 到 store 并写入本地存储 `registeredPhone = phone`。
4. 若接口返回 `isNew = true`（首次登录的新用户），跳转至实名认证页。
5. 否则跳转至首页 Tab。
6. 登录失败时展示错误信息（见 3.7 异常处理）。
7. 页面提供「忘记密码」链接，跳转至找回密码页。
8. 页面提供「立即注册」链接，跳转至注册页。

### 3.3.3 手机号 + 验证码登录

作为学员，我希望通过短信验证码免密登录，以便在忘记密码时快速登录。

验收条件：

1. 在登录页提供「验证码登录」Tab 或切换按钮。
2. 验证码登录表单包含：手机号（必填）、短信验证码（必填）。
3. 「获取验证码」触发 `POST /api/v1/auth/sms/send { phone, type: "login" }`，成功后按钮进入 60 秒倒计时。
4. 同一手机号 60 秒内不可重复发送；24 小时内最多 5 次；超出后提示（同 3.2.2 条款 5）。
5. 提交 `POST /api/v1/auth/phone/login-by-code { phone, code }`。
6. 成功后，保存 token 和 `registeredPhone`，跳转至首页或实名认证页（同 3.3.2）。
7. 失败时展示错误信息。

### 3.3.4 微信授权登录（微信小程序）

作为学员，我希望通过微信一键登录，以便快速访问小程序功能。

验收条件：

1. 登录页提供「微信登录」按钮。
2. 点击「微信登录」，调用 `Taro.login()` 获取 `code`，然后调用 `POST /api/v1/auth/wechat/login { code }`。
3. **首次微信登录（未绑定手机号）：** 后端返回 `phoneBindRequired = true` 和临时 token，前端弹出手机号绑定 Modal，用户填写手机号和验证码，调用 `POST /api/v1/auth/wechat/bind-phone { tempToken, phone, code }` 完成绑定。
4. **再次微信登录（已绑定手机号）：** 后端直接返回 `accessToken`，前端保存 token 跳转至首页。
5. 绑定成功后，将 token 和 `registeredPhone` 写入本地存储，跳转至首页或实名认证页。
6. 若手机号已被其他微信账号绑定，接口返回错误，提示：**该手机号已绑定其他账号**。

### 3.3.5 支付宝授权登录（支付宝小程序）

作为学员，我希望通过支付宝一键登录，以便快速访问小程序功能。

验收条件：

1. 登录页提供「支付宝登录」按钮。
2. 点击「支付宝登录」，调用 `Taro.login()` 获取 `code`，然后调用 `POST /api/v1/auth/alipay/login { code }`。
3. **首次支付宝登录（未绑定手机号）：** 后端返回 `phoneBindRequired = true` 和临时 token，前端弹出手机号绑定 Modal（同 3.3.4 条款 3）。
4. **再次支付宝登录（已绑定手机号）：** 后端直接返回 `accessToken`，前端保存 token 跳转至首页（同 3.3.4 条款 4）。
5. 绑定成功后跳转流程同 3.3.4 条款 5。
6. 若手机号已被其他支付宝账号绑定，提示同 3.3.4 条款 6。

## 3.4 找回密码

作为学员，我希望通过短信验证码重置密码，以便在忘记密码时恢复账号访问。

验收条件：

1. 表单包含：手机号（必填，11 位校验）、短信验证码（必填）、新密码（必填，≥ 6 位）。
2. 「获取验证码」触发 `POST /api/v1/auth/sms/send { phone, type: "reset" }`，成功后进入 60 秒倒计时。
3. 同一手机号 60 秒内不可重复发送；24 小时内最多 5 次；超出提示同 3.2.2 条款 5。
4. 提交 `POST /api/v1/auth/phone/reset-password { phone, code, newPassword }`，成功后 toast 提示：**密码重置成功**，1.5 秒后自动返回上一页。
5. 失败时展示接口返回的错误信息。
6. 页面提供「返回登录」链接，用户可随时返回登录页。

## 3.5 邀请关系管理与数据记录

作为平台，我希望准确记录学员的邀请关系链，以便支撑业绩追踪和提成计算。

验收条件：

1. 每个注册用户绑定的业务员信息存储至后端数据库，字段包括：
   - `userId`：学员 ID
   - `phone`：学员手机号
   - `inviteSalesmanId`：邀请业务员 ID
   - `inviteSalesmanName`：业务员名称（冗余存储用于展示）
   - `registeredAt`：注册时间
   - `sourceType`：来源类型（STAFF_CODE / STUDENT_INVITE）
2. 邀请关系在注册时一次性确定，后续不可修改。
3. 若注册时未提供 `inviteSalesmanId`（即没有归因参数），注册仍允许通过，但该用户不归属任何业务员，后端可将其标记为「无来源」或「直接注册」。
4. 已绑定的邀请关系不可被后续请求覆盖。

## 3.6 业务规则与限制

### 3.6.1 注册限制

| 规则                        | 说明                                           |
| --------------------------- | ---------------------------------------------- |
| ❌ 无机构码不可进入注册流程 | 注册首页强制机构码校验，校验失败不显示后续字段 |
| ❌ 禁止同一手机号重复注册   | 若手机号已注册，拦截并引导登录                 |
| ✅ 一个机构码可邀请无限用户 | 可在后台配置单个机构码的邀请上限（可选）       |
| ✅ 邀请关系在注册时确定     | 已绑定关系不可被后续请求覆盖                   |

### 3.6.2 登录限制

| 规则                     | 说明                                        |
| ------------------------ | ------------------------------------------- |
| ✅ 支持 4 种登录方式     | 手机号密码 / 验证码 / 微信 / 支付宝         |
| ⏱️ 连续登录失败 5 次以上 | 建议后端对该账号 IP 进行 15 分钟限流        |
| 🔐 微信 / 支付宝首次登录 | 强制绑定手机号，建立账号与微信/支付宝的关联 |
| 🔐 已绑定微信/支付宝     | 后续直接登录，无需重复绑定                  |

### 3.6.3 短信验证码限制

| 限制项                         | 阈值           | 处理                                   |
| ------------------------------ | -------------- | -------------------------------------- |
| 单手机号 60 秒内重复发送       | 不可重复       | 前端提示「请勿重复获取验证码」         |
| 单手机号 24 小时内发送次数     | 最多 5 次      | 超出拦截，提示「今日获取次数已达上限」 |
| 单手机号 24 小时内校验失败次数 | 最多 3 次      | 超出后冻结该手机号 1 小时              |
| 验证码有效期                   | 5 分钟         | 过期后需重新获取                       |
| 机构码校验失败                 | 24 小时 ≥ 5 次 | 冻结该手机号 30 分钟                   |

## 3.7 异常处理

### 3.7.1 注册异常

| 错误码 | HTTP 状态 | 提示文案                               | 用户操作                             |
| ------ | --------- | -------------------------------------- | ------------------------------------ |

## 3.8 非功能性需求

### 3.8.1 性能要求

| 指标               | 目标    | 说明               |
| ------------------ | ------- | ------------------ |
| 机构码校验响应时间 | ≤ 300ms | P95 响应时间       |
| 手机号登录响应时间 | ≤ 500ms | P95 响应时间       |
| 短信发送成功率     | ≥ 99%   | 依赖短信服务商 SLA |
| 页面加载时间       | ≤ 2s    | 首屏加载完成       |

### 3.8.2 兼容性

| 项               | 支持范围                 |
| ---------------- | ------------------------ |
| 小程序平台       | 微信小程序、支付宝小程序 |
| 微信小程序版本   | ≥ 7.0                    |
| 支付宝小程序版本 | ≥ 10.0                   |
| 操作系统         | iOS 12+、Android 6+      |

### 3.8.3 安全与合规

1. 注册、登录页面需展示《用户协议》《隐私政策》链接，用户点击可阅读完整内容。
2. 明确告知手机号用途：**该手机号用于账号注册、登录、找回密码和订单通知**。
3. 所有手机号、密码等敏感数据在前端需加密传输（HTTPS），后端不得明文存储密码（使用 bcrypt 等加密算法）。
4. 验证码存储至后端，前端无法直接访问或篡改。
5. 符合《个人信息保护法》《民法典》等法规要求。

### 3.8.4 可观测性

1. 注册、登录、验证码等关键链路必须记录审计日志，包括：时间戳、用户标识、操作类型、结果状态、错误码。
2. 关键指标需上报监控系统（打点）：
   - 注册转化漏斗（打开注册页 → 获取验证码 → 提交 → 注册成功）
   - 登录方式占比（密码 / 验证码 / 微信 / 支付宝）
   - 各错误码出现次数与频率

## 3.10 实名认证

作为平台，我希望在学员首次购课前完成实名认证，以便确保学员为成年人且具备完全民事行为能力。

验收条件：

1. 实名认证表单包含：真实姓名（必填）、身份证号（必填）、手机号码（必填）。
2. 三个字段均非空时方可提交。
3. 提交 `POST /api/v1/users/realname { name, idNo, phone }`。
4. 若后端判定学员年龄不满 18 周岁（通过身份证号 7-14 位解析出生年月计算），接口返回错误码 40001（UNDERAGE_USER），页面展示错误提示：未满 18 周岁，暂不可报名分期课程，提交按钮保持禁用。
5. 认证成功后 toast 提示：实名认证完成，并在 1.5 秒后跳转至首页 Tab。
6. 认证成功后，个人中心展示「✓ 已实名」标识。
7. 实名认证通过后，该功能入口在个人中心不再作为待完成提醒展示。

## 3.12 下单与签约

### 3.12.1 下单确认

作为学员，我希望在确认课程信息后提交订单，以便正式开始报名流程。

验收条件：

1. 下单确认页展示课程名称和所选付款方式说明：
   - 立即付款 → 一次性付款，立即开始学习
   - 先学后付 → 先学后付，每期课程结束后再扣款
2. 页面底部按钮文案：立即付款 → 「确认付款」；先学后付 → 「确认并签约」。

6. 提交先学后付需机构确认后，订单才生效，如机构认为学员不满足先学后付条件，可在订单生效前取消订单。
