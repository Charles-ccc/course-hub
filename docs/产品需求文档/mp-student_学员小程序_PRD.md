# mp-student · 学员小程序 PRD

---

# 1. 背景

学员小程序是「欢乐学」面向学员的核心消费端，同时支持微信小程序和支付宝小程序两个平台（基于 Taro 3 框架开发）。

学员通过业务员的邀请链接或二维码进入小程序，完成注册、实名认证、浏览课程、下单签约、分期还款和在线学习的完整消费旅程。「先学后付」是本产品最核心的商业模式：学员先学习本期课程，课程结束后系统自动代扣学费，降低学员的决策门槛。

## 1.1 功能范围

- 归因追踪：在应用启动时捕获来源（业务员/学员邀请），写入归因数据
- 注册与认证：手机号注册、实名认证（含未成年人拦截）、找回密码
- 课程发现：关键词搜索、课程列表、课程详情和免费试学
- 下单与签约：免费试学、创建订单、法大大电子签约、
- 学习：视频课播放、联系老师、人脸打卡获取学习激励
- 订单管理：查看订单列表、学费分期支付进度
- 个人中心：查看个人信息、实名认证入口、退出登录

## 1.2 价值

对学员：先体验后付款，降低购课决策风险；电子签约保障权益透明。  
对平台：归因追踪保障业务员提成准确计算；实名认证和合同签署构建合规基础。

# 3. 需求

## 3.1 应用启动与归因

作为平台，我希望在学员首次进入小程序时自动捕获来源归因，以便准确记录学员与业务员的绑定关系和学员互邀关系。

验收条件：

1. 应用启动时（onLaunch），解析启动参数中的 `?staff=<staffId>` 字段，若存在则写入本地存储键 `referrerStaffId`。
2. 启动参数中含 `?inv=<studentId>` 时，写入本地存储键 `referrerStudentId`。
3. 归因数据在后续注册请求时一并传入，其中 `referrerStaffId` 透传为 `inviteSalesmanId`；后端仅在首次注册成功时写入绑定关系，已绑定关系不可被后续请求覆盖。
4. 若已登录（token 有效），跳过引导页直接跳转至首页 Tab。
5. 若本地存储中存在 `registeredPhone`，尝试通过 Taro.login() 获取平台 authCode 并发起静默登录（`POST /api/v1/auth/alipay/login`），成功后跳转首页，失败后展示引导页。
6. 引导页提供「立即注册」和「已有账号，去登录」两个入口按钮。

---

## 3.2 手机号注册

作为学员，我希望通过手机号注册账号，以便开始使用平台购课。

验收条件：

**表单字段与校验**

1. 注册表单包含：手机号（必填）、短信验证码（必填）、密码（必填）、确认密码（必填）、机构码（必填）。
2. 点击「获取验证码」前，校验手机号是否为 11 位数字；不足 11 位时拦截并提示：请输入正确的手机号。
3. 点击「获取验证码」发送 `POST /api/v1/auth/sms/send { phone, type: "register" }`，成功后按钮进入 60 秒倒计时并禁用，倒计时结束后恢复为「获取验证码」。
4. 密码长度须 ≥ 6 位，不满足时提交拦截。
5. 确认密码与密码不一致时提交拦截，提示：两次密码输入不一致。
6. 机构码为必填项，为空时提交拦截。

**提交与结果** 7. 提交 `POST /api/v1/auth/phone/register { phone, code, password, orgCode, inviteSalesmanId }`，其中 `inviteSalesmanId` 来自本地存储的 `referrerStaffId`；成功后将 token 和 profile 存储至 Zustand store，同时写入本地存储 `registeredPhone = phone`。8. 注册成功后跳转至实名认证页（`/pages/auth/realname`），实名认证完成后方可购课。9. 注册失败（如验证码错误、机构码无效等）时，页面展示接口返回的错误信息。

---

## 3.3 手机号登录

作为学员，我希望用手机号和密码登录，以便继续我的学习进程。

验收条件：

1. 登录表单包含「手机号」和「密码」两个必填项，均非空时才允许提交。
2. 登录成功后，保存 token 到 store 并写入本地存储 `registeredPhone = phone`。
3. 若接口返回 `isNew = true`（首次登录的新用户），跳转至实名认证页。
4. 否则跳转至首页 Tab。
5. 登录失败时展示错误信息。
6. 页面提供「忘记密码」链接，跳转至找回密码页。
7. 页面提供「立即注册」链接，跳转至注册页。

---

## 3.4 找回密码

作为学员，我希望通过短信验证码重置密码，以便在忘记密码时恢复账号访问。

验收条件：

1. 表单包含：手机号（必填，11 位校验）、短信验证码（必填）、新密码（必填，≥ 6 位）。
2. 「获取验证码」触发 `POST /api/v1/auth/sms/send { phone, type: "reset" }`，成功后进入 60 秒倒计时。
3. 提交 `POST /api/v1/auth/phone/reset-password { phone, code, newPassword }`，成功后 toast 提示：密码重置成功，1.5 秒后自动返回上一页。
4. 失败时展示接口返回的错误信息。

---

## 3.5 实名认证

作为平台，我希望在学员首次购课前完成实名认证，以便确保学员为成年人且具备完全民事行为能力。

验收条件：

1. 实名认证表单包含：真实姓名（必填）、身份证号（必填）、手机号码（必填）。
2. 三个字段均非空时方可提交。
3. 提交 `POST /api/v1/users/realname { name, idNo, phone }`。
4. 若后端判定学员年龄不满 18 周岁（通过身份证号 7-14 位解析出生年月计算），接口返回错误码 40001（UNDERAGE_USER），页面展示错误提示：未满 18 周岁，暂不可报名分期课程，提交按钮保持禁用。
5. 认证成功后 toast 提示：实名认证完成，并在 1.5 秒后跳转至首页 Tab。
6. 认证成功后，个人中心展示「✓ 已实名」标识。
7. 实名认证通过后，该功能入口在个人中心不再作为待完成提醒展示。

## 3.6 课程发现

### 3.6.1 课程列表（首页）

作为学员，我希望浏览和搜索平台上的课程，以便找到适合自己的课程。

验收条件：

1. 首页顶部提供搜索框，支持输入关键词后点击搜索或键盘确认触发搜索。
2. 搜索或首次加载时请求 `GET /api/v1/courses?keyword=<关键词>&page=1&size=20`。
3. 课程卡片展示字段：课程名称、机构名称、每期价格（元/期，计算方式：price / periodCount / 100）、课程总价（元）、「先学后付」徽标。
4. 列表为空时展示空状态提示。
5. 未登录学员可浏览课程列表；点击课程卡片时，若未登录则跳转至登录页。
6. 已登录学员点击课程卡片跳转至课程详情页。

---

### 3.6.2 课程详情

作为学员，我希望在购课前了解课程详情并免费试学，以便做出更有把握的决策。

验收条件：

**页面内容**

1. 展示字段：课程名称、机构名称、课程总价（元）、每期金额（元）、分期期数、课程大纲（文本）、合规声明（平台为居间方，机构为合同甲方）。
2. 页面底部提供「免费试学第一节」按钮和「立即报名」按钮。
3. 底部提供付款方式选择：立即付款 / 先学后付（单选，默认「立即付款」）。

**免费试学** 4. 点击「免费试学第一节」，跳转至视频播放页，传入参数 `trial=1`，播放页仅展示第一节课程内容。

**购课确认** 5. 点击「立即报名」时，若用户尚未看过购课须知弹框（`trialConfirmed = false`），弹出购课须知 Modal，文案：购课须知：您已体验试学课程，购买后不以任何理由退课，请确认后提交。6. 学员在 Modal 内点击「确认」后，`trialConfirmed` 置为 true，跳转至下单确认页（携带 courseId 和 payType）。7. 若 `trialConfirmed` 已为 true，直接跳转至下单确认页，不再弹窗。8. `trialConfirmed` 为页面内本地状态，页面刷新后重置，不做持久化。

## 3.7 下单与签约

### 3.7.1 下单确认

作为学员，我希望在确认课程信息后提交订单，以便正式开始报名流程。

验收条件：

1. 下单确认页展示课程名称和所选付款方式说明：
   - 立即付款 → 一次性付款，立即开始学习
   - 先学后付 → 先学后付，每期课程结束后再扣款
2. 页面底部按钮文案：立即付款 → 「确认付款」；先学后付 → 「确认并签约」。
3. 点击确认按钮后，调用 `POST /api/v1/orders { courseId }`，成功后 toast 提示，1.5 秒后跳转至订单详情页。
4. 若后端返回错误码 40002（PRICE_LIMIT_EXCEEDED），提示：该课程暂不支持分期购买。
5. 若后端返回错误码 40001（UNDERAGE_USER），提示：未满 18 周岁，暂不可报名分期课程。
6. 提交先学后付需机构确认后，订单才生效，如机构认为学员不满足先学后付条件，可在订单生效前取消订单。

---

### 3.7.2 电子签约

作为学员，我希望在购课后通过电子合同签署完成协议，以便明确权利义务后进入正式学习阶段。

验收条件：

1. 订单详情页，当订单状态为 CREATED 时，操作区展示「签约授权（法大大）」按钮。
2. 点击签约按钮，调用 `POST /api/v1/orders/<orderId>/sign/fadadada`。
3. 签约成功后，页面内订单状态更新为 ACTIVE，签约按钮不再展示。
4. 签约成功后，系统自动生成分期计划，订单转为 ACTIVE 状态，学习正式开始。

## 3.8 视频学习

### 3.8.1 视频播放

作为学员，我希望在小程序内观看课程视频，以便完成学习任务。

验收条件：

1. 播放页加载时，请求 `GET /api/v1/courses/<courseId>/videos` 获取视频列表。
2. 点击视频列表中的某节课，调用 `GET /api/v1/courses/videos/<videoId>/url` 获取播放地址，并传入原生视频播放组件（Taro `<Video>`）。
3. 试学模式（URL 参数 `trial=1`）下，视频列表只展示第一节，且页面顶部展示黄色提示横幅：试学模式 · 仅展示第一节课程。
4. 试学模式下，页面底部展示报名引导卡片，提供购课入口。
5. 非试学模式下，完整展示所有视频列表，可自由切换节目。

### 3.8.3 人脸打卡

作为学员，我希望完成人脸验证打卡获取学习激励，以便抵扣部分学费。

验收条件：

1. 页面展示打卡说明和激励金额（¥5 /次，用于抵扣后续学费，金额可在管理后台设置）。
2. 点击「开始人脸验证」，调用平台人脸识别能力获取 faceToken（生产环境调用 wx.startFaceVerify 或 my.startFaceVerify，开发环境使用 mock faceToken）。
3. 获取 faceToken 后，调用 `POST /api/v1/learning/checkin { orderId, coursePeriod, faceToken }`。
4. 打卡成功时，页面展示验证通过状态，并显示本次获得的激励金额（如：学习激励已发放）。
5. 打卡失败（人脸比对不通过）时，页面展示验证失败状态，并提供「再次打卡」按钮。
6. 已完成打卡后，展示「再次打卡」按钮，允许多次打卡（具体次数限制由后端控制）。

## 3.9 订单管理

### 3.9.1 订单列表

作为学员，我希望查看我的所有购课记录，以便了解各课程的学习状态。

验收条件：

1. 进入订单列表页时，若未登录则跳转至登录页。
2. 加载 `GET /api/v1/orders`，展示全部订单，按创建时间倒序排列。
3. 每张订单卡片展示：课程名称、机构名称、订单状态、总金额（元）、分期期数、已付期数。
4. 订单状态标签配色：CREATED（待签约，琥珀色）、ACTIVE（学习中，绿色）、COMPLETED（已完成，灰色）、TERMINATED（已终止，灰色）。
5. 无订单时展示空状态提示，并提供「去看看课程」链接跳转至首页。
6. 点击订单卡片跳转至订单详情页。

### 3.9.2 订单详情与分期明细

作为学员，我希望查看订单的分期还款计划，以便了解每期到期时间和金额。

验收条件：

1. 订单详情页展示：课程名称、签约/收款方（机构名称）、订单总金额（元）、分期期数、每期金额（元）。
2. 订单状态为 ACTIVE 或 COMPLETED 时，展示分期还款计划列表。
3. 每条分期记录展示：期次（第 X 期）、到期日、金额（元）、状态标签。
4. 分期状态展示规则：PAID → 已还款；OVERDUE → 逾期；DELIVERED → 待扣款；PENDING → 待解锁。
5. 操作区根据订单状态展示对应操作：
   - CREATED → 「签约授权（法大大）」按钮
   - ACTIVE → 「去学习」按钮，点击跳转至学习中心
   - COMPLETED → 展示静态文字「课程已完成」，无按钮

## 3.10 个人中心

作为学员，我希望在个人中心查看账号信息并管理登录状态，以便了解认证情况和退出操作。

验收条件：

**已登录状态**

1. 展示头像（取真实姓名首字）、真实姓名、手机号。
2. 若 `profile.realnameVerified = true`，在姓名旁展示「✓ 已实名」徽标。
3. 若 `profile.realnameVerified = false`，展示「完成实名认证」菜单项（橙色警告样式），点击跳转至实名认证页。
4. 提供「我的课程」菜单项，点击跳转至订单列表页。
5. 「退出登录」按钮点击后，调用 store.logout() 清空 token，同时删除本地存储 `registeredPhone`，然后跳转至首页 Tab。

**未登录状态** 6. 未登录时，个人中心显示提示文字：登录后查看个人信息，并提供「去登录」按钮，点击跳转至登录页。7. 未登录时不展示头像、姓名、手机号等个人信息。
