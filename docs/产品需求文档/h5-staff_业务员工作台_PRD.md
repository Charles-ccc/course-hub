# h5-staff · 业务员工作台 PRD

---

# 1. 背景

业务员工作台是面向平台一线地推人员（业务员）的移动端 H5 应用，基于 React + Vite 开发，运行在微信/支付宝内置浏览器或普通手机浏览器中。

业务员是平台核心获客渠道，通过生成专属邀请二维码将潜在学员引导至学员小程序完成注册购课。每带入一笔签约或一次学员还款，系统生成对应提成记录。工作台需要支持他们完成「拉新 → 维护 → 对账」的完整日常工作闭环。

## 1.1 功能范围

- 账号体系：手机号/用户名 + 密码登录，仅允许管理后台预录入的业务员账号使用
- 邀请拉新：生成含业务员 ID 的专属二维码，学员扫码注册后自动绑定归属关系
- 学员跟进：查看名下学员列表，按还款日/逾期状态精准筛选
- 提成看板：分状态查看提成汇总（已结算/待结算/暂缓）及逐条提成明细

## 1.2 价值

对业务员：随时通过手机了解带单进展和提成状态，减少需要联系运营核对的频次。  
对平台：邀请链接与业务员 ID 强绑定，实现学员归属的自动化追踪，杜绝归属争议。

## 1.3 统一契约约定

1. 所有接口统一使用 `/api/v1` 前缀，鉴权方式统一为 `Authorization: Bearer <token>`。
2. 业务员端登录成功后，必须同时持久化 `staff_token`、`staff_id`、`staff_profile` 三个键；`staff_profile` 至少包含姓名、手机号、工号、合同类型、分组名称。
3. 鉴权页面进入时先校验 `staff_token` 与 `staff_id` 是否同时存在且 token 有效；若 token 失效则尝试刷新，刷新失败后清空会话并重定向登录页。
4. 金额字段统一按“分”传输，前端仅在展示层换算为“元”并保留两位小数；时间字段统一为 ISO 8601 UTC 字符串。
5. 学员列表与提成明细统一支持分页，默认 `pageSize=20`，切换筛选条件时重置为第一页。
6. 首页提成汇总仅展示已结算、待结算、暂缓三栏；已扣回仅在提成明细中展示，不计入首页汇总。
7. 业务员二维码由前端基于 `staff_id` 生成，接口只负责返回 profile 与统计数据，不额外提供二维码服务端接口。

# 3. 需求

## 3.1 登录

作为业务员，我希望通过手机号和密码登录工作台，以便查看我的学员和提成数据。

验收条件：

1. 登录表单包含「手机号/用户名」和「密码」两个必填项。
2. 手机号为空时，拦截提交并提示：请输入手机号。
3. 密码为空时，拦截提交并提示：请输入密码。
4. 点击「登录」后，按钮文字变为「登录中...」并禁用，防止重复提交。
5. 登录成功后，将 `staff_token`、`staff_id`、`staff_profile` 存入浏览器本地存储（localStorage），然后跳转至首页。
6. 登录失败时，页面显示错误提示：登录失败，请检查手机号或密码；或展示接口返回的错误消息。
7. 仅平台预先录入的业务员账号可登录，未录入的手机号提交后返回失败。
8. 所有需要鉴权的页面在加载时检查本地存储中的 `staff_token` 与 `staff_id`，若任一缺失则重定向至登录页。
9. 登录页底部显示说明文字：仅限已审核的业务员使用。

---

## 3.2 首页

作为业务员，我希望在工作台首页快速看到当前收入概况并获取邀请链接，以便开始今日的拉新工作。

验收条件：

**提成汇总卡片**

1. 首页展示三张提成统计卡片：已结算提成（元）、待结算提成（元）、暂缓/逾期中（元）。
2. 金额由分换算为元后展示，保留两位小数。
3. 数据通过 `GET /api/v1/staff/commission` 在页面挂载时自动拉取，加载期间卡片显示占位状态。
4. 汇总接口只统计 `SETTLED`、`PENDING`、`HELD` 三类金额，`CLAWED_BACK` 不进入首页汇总。

**专属邀请二维码** 4. 展示该业务员的专属邀请二维码，二维码内容为 `{TARO_APP_SHARE_URL}/register?staff={staffId}`，其中 `TARO_APP_SHARE_URL` 默认为 `https://mp.wangke.com`。5. 二维码由客户端使用 qrcode 库实时生成，无需服务端接口。6. 若二维码生成库不可用（如在小程序 webview 环境中），降级展示邀请链接纯文本。7. 二维码卡片下方提供「复制链接」按钮，点击后将邀请链接写入设备剪贴板，按钮文字即时变为「已复制」，2 秒后恢复为「复制链接」。8. 卡片说明文字：扫码后学员将自动绑定至您名下。

**快捷导航** 9. 首页提供「我的学员列表」和「提成明细」两个导航入口，点击分别跳转至对应页面。

---

## 3.3 学员列表

作为业务员，我希望查看我名下的学员列表并按还款节点筛选，以便精准定位需要关注的学员。

验收条件：

**筛选 Tab**

1. 页面顶部提供四个筛选 Tab：全部 / 7天还款日 / 还款日 / 已逾期。
2. 各 Tab 过滤逻辑：
   - 全部：展示所有学员，不过滤
   - 7天还款日：学员最近一期 nextDueDate 与今日相差 0～7 天（含今日）
   - 还款日：nextDueDate 在今日前后 24 小时内
   - 已逾期：学员订单状态为 OVERDUE
3. 切换 Tab 时，重新发送请求 `GET /api/v1/staff/students?page=1&tab={tab}`，tab 值分别为 `all`、`due7`、`due`、`overdue`。
4. 请求期间列表区域显示「加载中...」提示。
5. 接口返回的学员记录必须包含 `nextDueDate`、`overdueDays`、`progressFinishedCount`、`progressTotalCount` 等派生字段，前端不自行反推还款状态。

**学员卡片内容** 5. 每张学员卡片展示：脱敏姓名（姓氏 + _，如「林_」）、脱敏手机号（如「138\*\*\*\*1234」）、订单状态标签、学习进度（已学完 X/N 期）、还款日提示标签。6. 订单状态标签配色：ACTIVE（学习中，绿色）；COMPLETED（已完成，蓝色）；OVERDUE（已逾期，红色）。7. 还款日提示标签显示规则：

- nextDueDate 早于今日 → 「逾期 N 天」（红色标签），N 为逾期天数
- nextDueDate 为今日 → 「今日还款」（黄色标签）
- nextDueDate 在 1～7 天后 → 「N 天后还款」（黄色标签）
- nextDueDate 在 7 天后以上 → 不展示标签

**空状态** 8. 「全部」Tab 无数据时显示：暂无学员。9. 「已逾期」Tab 无数据时显示：暂无逾期学员。10. 其他 Tab 无数据时显示：暂无符合条件的学员。

**限制** 11. 学员卡片为只读展示，当前版本无点击跳转至学员详情的操作。

## 3.4 提成明细

作为业务员，我希望查看每笔提成的来源和状态，以便核对收入是否准确。

验收条件：

**提成汇总**

1. 页面顶部以三栏横排展示：已结算（绿色）、待结算（琥珀色）、暂缓（蓝色），金额单位为元，保留两位小数。
2. 数据与首页统计卡片来源相同（均来自 `GET /api/v1/staff/commission`）。

**提成明细列表** 3. 列表按创建时间倒序展示每条提成记录，每条包含：提成类型、关联课程/学员、金额（元）、发生日期、状态标签。4. 提成类型显示规则：

- type = CLOSING → 成单提成
- type = PERFORMANCE → 履约提成（第 N 期），N 为 periodNo 字段值

5. 状态标签显示规则：
   - PENDING → 待结算（琥珀色）
   - SETTLED → 已结算（绿色）
   - HELD → 暂缓（蓝色）
   - CLAWED_BACK → 已扣回（红色）
6. 列表为只读展示，无操作入口。
7. 列表分页默认 `pageSize=20`，按照创建时间倒序分页加载。

---

## 3.5 个人中心

作为业务员，我希望在个人中心查看账号信息并退出登录，以便管理我的工作台登录状态。

验收条件：

1. 页面展示：头像（取姓名首字显示，无上传功能）、姓名、工号（staff_id）、手机号、合同类型。
2. 合同类型显示规则：contractType = EMPLOYEE → 员工制；其他 → 代理制。
3. 若 groupName 字段不为空，在合同类型后拼接显示「· {groupName}」，如「代理制 · 华东增长组」。
4. 页面提供「我的学员」和「提成明细」菜单入口，点击使用 navigateTo 跳转（不使用 switchTab，因为这两个页面不在 TabBar 中）。
5. 点击「退出登录」后，清除本地存储中的 staff_token、staff_id、staff_profile 三个键，然后使用 redirectTo 跳转至登录页（不保留页面栈）。
6. 页面挂载时请求 `GET /api/v1/staff/profile`，并将最新 profile 数据更新至本地存储。

## 3.6 业务状态与数据口径

1. 业务员登录态仅由 `staff_token` 决定，但页面级权限判断必须同时校验 `staff_id` 是否存在。
2. `GET /api/v1/staff/profile` 的返回值是个人中心的唯一数据源，若与本地存储不一致，以接口返回为准并覆盖本地存储。
3. 业务员二维码链接固定为 `{TARO_APP_SHARE_URL}/register?staff={staffId}`，默认域名为 `https://mp.wangke.com`。
4. 学员注册时若携带 `staff` 归因参数，后端仅在首次绑定时写入归属关系，已绑定关系不可被后续请求覆盖。
5. 仅状态为 `ACTIVE` 的业务员可被用于新的归因绑定与邀请链接注册；`DISABLED` 业务员的历史数据保留，但禁止继续新增归因。
